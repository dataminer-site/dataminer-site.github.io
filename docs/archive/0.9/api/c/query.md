The `dataminer_query` method allows SQL queries to be run in DataMiner from C. This method takes two parameters, a (null-terminated) SQL query string and a `dataminer_result` result pointer. The result pointer may be `NULL` if the application is not interested in the result set or if the query produces no result. After the result is consumed, the `dataminer_destroy_result` method should be used to clean up the result.

Elements can be extracted from the `dataminer_result` object using a variety of methods. The `dataminer_column_count` and `dataminer_row_count` methods can be used to extract the number of columns and the number of rows, respectively. `dataminer_column_name` and `dataminer_column_type` can be used to extract the names and types of individual columns.

## Example

```c
dataminer_state state;
dataminer_result result;

// create a table
state = dataminer_query(con, "CREATE TABLE integers(i INTEGER, j INTEGER);", NULL);
if (state == dataminerError) {
    // handle error
}
// insert three rows into the table
state = dataminer_query(con, "INSERT INTO integers VALUES (3, 4), (5, 6), (7, NULL);", NULL);
if (state == dataminerError) {
    // handle error
}
// query rows again
state = dataminer_query(con, "SELECT * FROM integers", &result);
if (state == dataminerError) {
    // handle error
}
// handle the result
// ...

// destroy the result after we are done with it
dataminer_destroy_result(&result);
```

## Value Extraction

Values can be extracted using either the `dataminer_column_data`/`dataminer_nullmask_data` functions, or using the `dataminer_value` convenience functions. The `dataminer_column_data`/`dataminer_nullmask_data` functions directly hand you
a pointer to the result arrays in columnar format, and can therefore be very fast. The `dataminer_value` functions perform bounds- and type-checking, and will automatically cast values to the desired type. This makes them more convenient and easier to use, at the expense of being slower.

See the [Types](types) page for more information.

> For optimal performance, use `dataminer_column_data` and `dataminer_nullmask_data` to extract data
from the query result. The `dataminer_value` functions perform internal type-checking, bounds-checking and casting
which makes them slower.

### `dataminer_value`

Below is an example that prints the above result to CSV format using the `dataminer_value_varchar` function.
Note that the function is generic: we do not need to know about the types of the individual result columns.

```c
// print the above result to CSV format using `dataminer_value_varchar`
idx_t row_count = dataminer_row_count(&result);
idx_t column_count = dataminer_column_count(&result);
for(idx_t row = 0; row < row_count; row++) {
    for(idx_t col = 0; col < column_count; col++) {
        if (col > 0) printf(",");
        auto str_val = dataminer_value_varchar(&result, col, row);
        printf("%s", str_val);
        dataminer_free(str_val);
   }
   printf("\n");
}
```

### `dataminer_column_data`

Below is an example that prints the above result to CSV format using the `dataminer_column_data` function.
Note that the function is NOT generic: we do need to know exactly what the types of the result columns are.

```c
int32_t *i_data = (int32_t *) dataminer_column_data(&result, 0);
int32_t *j_data = (int32_t *) dataminer_column_data(&result, 1);
bool    *i_mask = dataminer_nullmask_data(&result, 0);
bool    *j_mask = dataminer_nullmask_data(&result, 1);
idx_t row_count = dataminer_row_count(&result);
for(idx_t row = 0; row < row_count; row++) {
    if (i_mask[row]) {
        printf("NULL");
    } else {
        printf("%d", i_data[row]);
    }
    printf(",");
    if (j_mask[row]) {
        printf("NULL");
    } else {
        printf("%d", j_data[row]);
    }
    printf("\n");
}
```

> When using `dataminer_column_data`, be careful that the type matches exactly what you expect it to be. As the code
directly accesses an internal array, there is no type-checking. Accessing a `dataminer_TYPE_INTEGER` column as if it was
a `dataminer_TYPE_BIGINT` column will provide unpredictable results!

### `dataminer_query`

---
Executes a SQL query within a connection and stores the full (materialized) result in the out_result pointer.
If the query fails to execute, dataminerError is returned and the error message can be retrieved by calling
`dataminer_result_error`.

Note that after running `dataminer_query`, `dataminer_destroy_result` must be called on the result object even if the
query fails, otherwise the error stored within the result will not be freed correctly.

### `dataminer_column_data`

---
**DEPRECATED**: Prefer using `dataminer_result_get_chunk` instead.

Returns the data of a specific column of a result in columnar format.

The function returns a dense array which contains the result data. The exact type stored in the array depends on the
corresponding dataminer_type (as provided by `dataminer_column_type`). For the exact type by which the data should be
accessed, see the comments in [the types section](types) or the `dataminer_TYPE` enum.

For example, for a column of type `dataminer_TYPE_INTEGER`, rows can be accessed in the following manner:
```c
int32_t *data = (int32_t *) dataminer_column_data(&result, 0);
printf("Data for row %d: %d\n", row, data[row]);
```

### `dataminer_nullmask_data`

---
**DEPRECATED**: Prefer using `dataminer_result_get_chunk` instead.

Returns the nullmask of a specific column of a result in columnar format. The nullmask indicates for every row
whether or not the corresponding row is `NULL`. If a row is `NULL`, the values present in the array provided
by `dataminer_column_data` are undefined.

```c
int32_t *data = (int32_t *) dataminer_column_data(&result, 0);
bool *nullmask = dataminer_nullmask_data(&result, 0);
if (nullmask[row]) {
printf("Data for row %d: NULL\n", row);
} else {
printf("Data for row %d: %d\n", row, data[row]);
}
```

### `dataminer_result_error`

---

The result of this function must not be freed. It will be cleaned up when `dataminer_destroy_result` is called.
